处理逻辑 点击展开查看 6 步流程

1解析 JSONL

按行读取，每行一条 trace 记录；提取 datasetItemId、messages、baseline_model_response、traceOutput、scores。

2按 item 聚合

同一 datasetItemId 可能有多条记录，优先挑有 traceOutput 的那条；顺序按首次出现固定。

3字段提取

query 取 messages 最后一条的纯文本（跳过 resource / image 标签）；eval_answer 取 final_output.content 里 type=text 部分；thinking 取 type=think/thinking/reasoning 部分（或 final_output.thinking / reasoning_content 兜底）。内容按原样保留，不截断也不做额外处理。

4代码符号清洗

baseline_answer / eval_answer / thinking 剥 ``` 代码块围栏、反引号、# 标题、**粗体** 等 Markdown，保留排版与换行。

5分数展开

自动识别：多配置（每个评测维度两列：分数 + 理由，列数按文件实际出现的 score 动态）或单配置（score_config/value/comment）。

6长内容直出

Excel 规格单元格上限 32767 字符；本工具已解除 SheetJS 的硬校验，超长也原样写入 xlsx。Excel / WPS 的显示表现取决于各自实现。

输出列说明 两种 schema 的列布局

两种模式都会生成一行一 item（按源文件里 datasetItemId 首次出现顺序编号）。没有 traceOutput 的 item 会保留 query / key_point，分数列留空。

多配置评测 · 动态列数
  • 序号 / datasetItemId / query / baseline_answer / eval_answer / thinking / key_point
  • 下游分数列按文件实际出现的 score.name 动态拼接，每个两列：[xxx] 分数 / [xxx] 理由
  • 已知顺序（优先）：online-单向-truthful / if / reasoning / perception、Vision-限制性指令、Vision-AI生图识别、底线-Repeat / 不正常输出代码 / 语言杂糅 / 不合理拒答
  • 文件里出现的其他新 score 名称会自动追加在已知列后面
